# 07-qc-validate: 품질 검증 스킬

> **실행자**: Leader (직접 실행)
> **실행 시점**: T5 완료 후
> **쓰기 권한**: `output/`, `quarantine/`, PIPELINE-LOG.md

## 목적

생성된 영상 파일이 배포 기준(포맷, 해상도, 안전성)을 모두 충족하는지 최종적으로 교차 검증합니다.
불필요한 메타데이터와 API Key 흔적 등을 안전하게 삭제하여 유출을 방지합니다.

## 실행 시점

- 06-edit-compose (T5)의 `composed.mp4` 렌더링이 완료된 후 배포 직전에 자동 활성화됩니다.
- 유저에게 전달할 영상물의 무결성을 검사하고 안전 조치를 취할 때 사용합니다.

## 워크플로우

### Step 1: 파일 무결성 체크

- `work/T5-edit/composed.mp4`의 존재 및 크기를 확인하여 정상적인 파일 구조(1MB ~ 5GB 범주)인지 검사합니다.
- 비디오(H.264), 오디오(AAC), 해상도(1080p), FPS, 길이 오차 등을 `ffprobe`로 스캔합니다.

### Step 2: 안전성(NSFW/가이드라인) 검사

- 영상에서 구간별로 10개 프레임을 균등 샘플링하여 비전 모델이나 안전 검사 도구를 통해 스캔합니다.
- 위반 사항(NSFW 등) 감지 시 해당 영상을 폐기하지 않고 `quarantine/` 디렉토리에 격리 조치시킵니다.

### Step 3: 메타데이터 스트립(살균) 처리

- ffmpeg 등을 사용해 시스템 경로, 로컬 식별자, MCP 서버 관련 메타데이터, GPS 정보, EXIF 태그 등 불필요한 모든 메타데이터를 클리어(`-map_metadata -1`)합니다.

### Step 4: 시크릿 침해 검사

- 최종 파일 바이너리에서 API 키(sk-, pk- 형태) 텍스트나 보안과 직결된 시스템 프롬프트 조각이 파일 스트림에 잔류하는지 스캔합니다.
- 발견 즉시 `security.log`에 남기고 재스크립(Re-strip)을 시도합니다.

### Step 5: 배포 결식 반영

- 모든 검증을 통과 시 `output/final.mp4` 경로로 안전하게 복사하고 `FINAL-REPORT.md`를 함께 생성(작성)합니다.
- 패스하지 못했으나 복구 불가한 에셋은 `quarantine/`으로 옮기고 실패 내역을 파이프라인 로그에 표기한 뒤 유저에게 알립니다.

1. 해상도/코덱 1차 미충족 시: 즉시 불합격 처리하지 않고, 빠른 폴백 컨버팅(ffmpeg)으로 재인코딩(1회 한정)을 거친 후 재검사합니다.
2. 단기 프레임 샘플링 안전성 애매함: 10개 프레임 스캔 결과가 확신할 수 없는 단계인 경우 Leader의 정성 평가로 통과 여부를 묻습니다.
