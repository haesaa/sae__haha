## Phase 1: 분석 (T1)

**담당**: Analyzer 팀원

### P1 (MV Mode)

1. 음악 파일을 분석하여 BPM, 구간, 감정을 추출
2. 장면별 프롬프트를 자동 생성
3. 유저에게 분석 요약 리포트 제출

### P2 (Scene Mode)

1. 시놉시스 파싱 + 콘티 3x3 그리드 분리
2. 캐릭터시트에서 인물 특징 추출
3. 장면별 프롬프트 생성

### P3 (Animation Mode)

1. 원본 영상 프레임 추출 + 모션 분석
2. 참조이미지 스타일 벡터 추출
3. 스타일 트랜스퍼 프롬프트 생성

**산출물**: `work/T1-analysis/analysis.json`, `prompts/`

---

## Phase 2: 이미지 생성 (T2)

**담당**: Creator 팀원

1. **Plan Approval**: 테스트 이미지 1장 생성 → Leader 자동 검증
   - 해상도 ≥ 1024x1024
   - 캐릭터 유사도 ≥ 0.8 (시트 있는 경우)
   - 미달 → 프롬프트 수정 → 재생성 (최대 3회)
2. 승인 → 나머지 장면 일괄 생성
3. 유저에게 3x3 시트 또는 장면 이미지 제출

**산출물**: `work/T2-images/scene-{NN}.png`

---

## Phase 3: 모션/영상 생성 (T3)

**담당**: Creator 팀원 (T2 완료 후)

1. 장면별 img2vid 변환
2. 길이: 구간별 3~6초
3. 실패 복구: 동일 MCP 재시도 → 폴백 MCP → Ken Burns 대체
4. **한 장면 실패해도 나머지 계속 진행**
5. 유저에게 전체 클립 미리보기 제출

**산출물**: `work/T3-clips/scene-{NN}.mp4`

---

## Phase 4: 오디오 (T4) — T2와 병렬

**담당**: Editor 팀원

- P1: 원본 음악 복사 (별도 생성 불필요)
- P2: BGM 생성 + 내레이션 생성
- P3: 원본 오디오 추출 보존

**산출물**: `work/T4-audio/bgm.mp3`, `narration-{NN}.mp3`

---

## Phase 5: 편집/합성 (T5)

**담당**: Editor 팀원 (T3+T4 완료 후)

1. 비트 싱크 기반 타임라인 구성
2. 트랜지션 적용 (모드별 전략)
3. 오디오 믹싱
4. 최종 렌더링: 1920x1080 H.264 AAC
5. 유저에게 완성본 제출

**산출물**: `work/T5-edit/composed.mp4`

---

## 중간 확인 포인트 (★)

| 시점 | 내용 | 유저 액션 |
|------|------|----------|
| T1 완료 후 | 분석 요약 (BPM, 구간, 감정) | 승인/수정 |
| T2 완료 후 | 생성된 이미지 | 승인/수정/재생성 |
| T3 완료 후 | 전체 클립 미리보기 | 승인/수정 |
| T5 완료 후 | 완성본 | 최종 승인 |

유저 피드백 → 해당 Phase 재실행 or 다음 Phase 진행

## 에러 복구 전략

| 실패 유형 | 1차 시도 | 2차 시도 | 3차 시도 | 최종 폴백 |
|----------|---------|---------|---------|----------|
| MCP 호출 실패 | 동일 MCP 재시도 | 2순위 MCP | 3순위 MCP | 유저 알림 |
| 품질 미달 | 프롬프트 수정 | 다른 모델 | 파라미터 조정 | 유저 수동 개입 |
| 타임아웃 | 5s 대기 후 재시도 | 15s 대기 | 45s 대기 | 폴백 MCP |
| 전체 실패 | - | - | - | Graceful Degradation |

## 재시도/복구 방법

유저가 이전 프로젝트를 이어서 작업하려면:

```
"project-{timestamp} T{N}부터 재실행해줘"
```

- `work/T{N}/` 폴더가 체크포인트 역할
- 이미 완료된 단계는 자동 스킵
- PIPELINE-LOG.md에 재실행 기록 추가
