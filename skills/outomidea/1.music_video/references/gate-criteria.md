# Gate 통과 기준 체크리스트

> 각 Phase Gate의 정량/정성 판정 기준. `config/quality-gates.json`의 상세 해설.

---

## Gate-T1: 분석 결과 검증

### 자동 검증 (모든 모드 공통)

- [ ] `analysis.json` 파일 존재 + 크기 > 0
- [ ] 프롬프트 파일 수 = 장면 수

### P1 (MV) 추가 기준

- [ ] BPM ∈ [40, 220] (이 범위 밖이면 분석 오류 의심)
- [ ] 구간(sections) ≥ 3개 (최소 intro + body + outro)
- [ ] 비트 타이밍 배열(beat_times) 비어있지 않음
- [ ] 에너지 곡선 데이터 포인트 ≥ 10개

### P2 (Scene) 추가 기준

- [ ] 장면 수 ≥ 콘티 프레임 수
- [ ] 각 장면에 필수 필드 존재 (scene_id, action, camera)
- [ ] 캐릭터 참조 경로가 실제 파일과 매칭

### P3 (Animation) 추가 기준

- [ ] 키프레임 추출 완료 (최소 10장)
- [ ] 스타일 벡터 생성 완료

### 정성 리뷰 (★ 중간확인 #1)

- 유저에게 분석 요약 제출
- 구간 분리가 음악 구조와 일치하는지 확인

---

## Gate-T2: 이미지 품질 검증

### Plan Approval (사전 게이트)

- [ ] 테스트 이미지 해상도 ≥ 1024x1024
- [ ] 캐릭터 유사도 ≥ 0.8 (캐릭터시트 제공 시)
- [ ] 프롬프트 무드와 이미지 분위기 매칭
- 미달 → 프롬프트 수정 후 재생성 (최대 3회)

### 자동 검증 (일괄 생성 후)

- [ ] 모든 이미지 해상도 ≥ 1024x1024
- [ ] 생성된 이미지 수 = analysis.json 장면 수
- [ ] 파일 형식: PNG 또는 JPG
- [ ] 모든 파일 크기 > 0

### 정성 리뷰 (★ 중간확인 #2)

- 장면 간 스타일 일관성
- 캐릭터 외형의 장면 간 일관성
- 배경과 조명이 무드와 매칭

---

## Gate-T3: 영상 클립 검증

### 자동 검증

- [ ] 각 클립 길이 오차 ≤ ±0.5초 (목표 길이 기준)
- [ ] 해상도 = T2 이미지와 일치
- [ ] 코덱: H.264 재생 가능
- [ ] 모든 파일 크기 > 0

### 정성 리뷰 (★ 중간확인 #3)

- 캐릭터 변형이 과도하지 않은지
- 모션이 자연스러운지 (부자연스러운 워핑, 깜빡임 없음)
- 카메라 무브먼트가 프롬프트 지시와 일치

---

## Gate-T4: 오디오 검증

### 자동 검증

- [ ] 오디오 파일 최소 1개 존재 (bgm 또는 original)
- [ ] 모든 파일 크기 > 0
- [ ] 코덱: MP3/AAC/WAV 디코드 가능
- [ ] 길이 오차:
  - P1: = 원본 음악 길이 ±1초
  - P2: ≥ 전체 영상 길이의 80%

---

## Gate-T5: 합성 영상 검증

### 자동 검증

- [ ] 비트 싱크 오차 ≤ 50ms
- [ ] 총 길이 = 원곡/시놉시스 기준 ±1초
- [ ] 해상도: 1920x1080
- [ ] 코덱: H.264 (비디오) + AAC (오디오)
- [ ] 파일 크기 > 1MB

### 정성 리뷰 (★ 최종확인)

- 전체 영상 정상 재생 (끊김, 검은 프레임 없음)
- 트랜지션이 자연스러운지
- 오디오-비디오 싱크 확인

---

## Gate-QC: 최종 품질/보안 검증

### 파일 무결성

- [ ] `composed.mp4` 존재 + 크기 > 0
- [ ] 크기 범위: 1MB ~ 5GB

### 보안 스캔

- [ ] NSFW: 10프레임 균등 샘플링 → 모두 안전
- [ ] 메타데이터: EXIF/GPS/인코더 정보 완전 제거
- [ ] API 키 잔류: sk-/pk-/key- 접두어 미감지
- [ ] 프롬프트 텍스트 잔류: 미감지

### 통과 → output/

- `output/final.mp4` 복사
- `output/final.sha256` 해시 생성
- `output/FINAL-REPORT.md` 작성
